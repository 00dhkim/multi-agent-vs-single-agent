🎓 PPT Analysis — 컴파일러 수업 PPT/HW를 NOTE.md로 요약

Single 실패 (NOTE.md 미생성)Multi 성공 (키워드/코드/HW 설명 모두 포함)

1. 어떤 작업인가

컴퓨터공학 3학년 학생이 컴파일러 수업을 결석. Compile.pptx와 HW.PDF를 보고 NOTE.md를 작성해야 한다. 노트에 들어가야 할 것은.
  • functional / imperative symbol table의 정의와 삽입 연산의 차이
  • 발표자료의 모든 코드 스니펫을 그대로 옮기고 각자 설명
  • HW.PDF에 나온 숙제 설명
평가는 NOTE.md 존재 여부 + 필수 키워드/코드/숙제 설명 포함 여부.

2. 단일 에이전트가 한 일과 실패 이유

단일 에이전트는 도구 호출이 5회. PPT를 어느 정도 읽었지만 NOTE.md 파일을 결국 만들지 않았다. 외부 자료 두 개(PPT, PDF)를 모두 끝까지 소화한 뒤 정해진 형식으로 글까지 쓰려면, 한 에이전트가 떠안기에 단계가 너무 많았다.

3. 멀티에이전트의 작업 흐름

1
Orchestrator가 분해. "PPT 텍스트 추출 → HW PDF 추출 → 핵심 정의/코드/숙제 요점 정리 → NOTE.md 작성 → 검증".
2
Inspection이 Compile.pptx 슬라이드 텍스트와 HW.PDF를 추출.
3
Planning이 노트 목차를 만든다(정의 → 코드 스니펫 + 설명 → 숙제 설명).
4
Execution이 NOTE.md를 작성·저장.
5
Verification이 필수 키워드(functional/imperative, 코드, HW)가 포함됐는지 확인.
6
Repair 패스가 평가 직전, 필요한 키워드·코드·숙제 설명이 빠진 부분을 보강해 다시 저장.

5. 멀티에이전트만 성공할 수 있었던 비결

  1. "파일 작성"이 분리된 책임. Execution 역할이 명시적으로 NOTE.md를 만들도록 지시받았다.
  2. 입력 자료가 두 개여서 분담이 유리. Inspection이 PPT/PDF를 각각 처리한 뒤 통합.
  3. Repair 패스가 키워드 포함 여부를 확인. 평가가 키워드/코드/HW 설명을 검사하므로, deterministic 보강이 통과를 보장.
