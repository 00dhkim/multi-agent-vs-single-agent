📊 Excel Data Transformation — 2차원 표를 1차원 레코드로

Single 실패 (Processed.xlsx 미생성)Multi 성공

1. 어떤 작업인가

Household_Appliances.xlsx에는 냉장고·에어컨·세탁기 매출이 가로로 펼쳐진 2차원 표로 들어 있다. 이것을 Format_Example.xlsx에 예시된 1차원 레코드(한 행 = 한 시점·한 품목) 형식으로 변환해 Processed.xlsx로 저장해야 한다.
  • 2024년 12월부터 거꾸로 정렬
  • 데이터가 하나라도 있는 월만 포함
  • 알 수 없는 필드는 공란

2. 단일 에이전트가 한 일과 실패 이유

단일 에이전트는 원본 파일을 열어보고 형식 예시도 확인했지만, 결과 파일 Processed.xlsx를 만드는 마지막 단계에서 그냥 종료했다. 평가는 산출 파일 존재 + 데이터 정확도를 모두 검사하므로 0점.

전형적인 "조사는 했는데 산출물을 만들지 않음" 실패 모드.

3. 멀티에이전트의 작업 흐름

1
Orchestrator가 분해. "입력 워크북 구조 분석 → 예시 워크북 구조 분석 → 매핑 규칙 도출 → 변환 → 파일 저장 → 검증".
2
Inspection이 Household_Appliances.xlsx와 Format_Example.xlsx를 각각 따로 읽어 컬럼/시트/예시 행을 추출.
3
Planning이 "원본의 어떤 컬럼이 목표의 어떤 필드에 대응하는가" 표를 만든다.
4
Execution이 실제 1차원 레코드로 변환하고 Processed.xlsx를 저장.
5
Verification이 행 수와 역순 정렬, 빈 셀 규칙 확인.
6
Repair 패스가 평가 직전 파일 존재와 형식 일치를 다시 확인.

5. 멀티에이전트만 성공할 수 있었던 비결

  1. 입력과 예시를 분리해서 보기. Inspection이 두 파일을 각각 분석해 매핑 규칙을 명확히 했다.
  2. "산출 파일을 만드는 일"이 별도 단계. Execution의 책임이 분명하니 "조사만 하고 끝나는" 실수가 사라졌다.
  3. Repair 패스의 마지막 점검. 평가 직전 파일 존재 확인이 한 번 더 들어가 가장 흔한 실패 모드를 차단.
